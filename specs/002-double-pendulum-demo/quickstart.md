# Quickstart: 二重振り子シミュレーションデモページ

**Date**: 2026-06-11 | **Plan**: [plan.md](./plan.md)

実装完了後にエンドツーエンドで動作検証するための手順。

## 前提

- pnpm がインストール済み、依存関係は `pnpm install` 済み
- ブランチ: `002-double-pendulum-demo`

## 起動

```bash
pnpm dev
# → http://localhost:3000/examples/simulation/double-pendulum
```

## 検証シナリオ

### S1: カタログ導線（US1 / FR-001）

1. `http://localhost:3000/catalog` を開く
2. サイドバーに「シミュレーション」カテゴリが表示される
3. 「二重振り子」カードからデモページへ遷移できる

### S2: シミュレーション再生（US1 / FR-002, FR-005）

1. デモページで開始ボタンを押す → 振り子が揺れ始め、不規則な軌道を描く
2. 一時停止 → その場で静止。再開 → 続きから動く
3. リセット → 初期位置に戻り、光跡が消える

### S3: パラメータ調整（US2 / FR-003, FR-004, FR-009）

1. 節 1 の長さスライダーを最大にしてリセット → 上腕が長くなる
2. 開始角度スライダーを変えてリセット → 指定角度から動き出す
3. 同じ設定でリセットを 10 回繰り返す → 毎回同じ動き出しになる（SC-005）

### S4: 光跡（US3 / FR-006）

1. 再生中、先端の軌跡が光る尾として残り、数秒で滑らかに消えていく
2. 一時停止中は光跡が消えずに保持される
3. 5 分間放置しても滑らかさが維持される（SC-004。光跡が無限に濃くならない）

### S5: テーマ・レスポンシブ・アクセシビリティ（FR-008 / Edge Cases）

1. ダーク/ライトモードを切り替え → 配色が破綻しない
2. ブラウザ幅をモバイル相当（375px）に縮小 → 描画領域とコントロールが重ならない
3. OS の「視差効果を減らす」を有効にしてリロード → 自動再生されない
4. Tab キーでスライダー・ボタンへフォーカス移動と操作ができる

## 自動テスト・品質ゲート（憲法 II）

```bash
pnpm lint && pnpm type-check && pnpm test:run   # すべて成功すること
pnpm build                                       # ビルドが成功すること
pnpm storybook                                   # PendulumControls / Slider のストーリー確認
```

ユニットテストの要点は [contracts/components.md](./contracts/components.md) の
「契約上の保証」を参照（決定論性・純関数性・エネルギー保存・クランプ）。

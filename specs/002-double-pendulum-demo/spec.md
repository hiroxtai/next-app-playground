# Feature Specification: 二重振り子シミュレーションデモページ

**Feature Branch**: `002-double-pendulum-demo`

**Created**: 2026-06-11

**Status**: Draft

**Input**: User description: "学習用途から少し外れるが、Next.js を使用したゲームや物理シミュレーションのデモページを作成したい。まずは、二重振り子の挙動を再現するデモページを作成したい。二重振り子の節の長さや、開始位置を調整できるようにしてほしい。また、振り子の先端の光跡をしばらく残すような表示にして見た目が美しい表示にしてほしい。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 二重振り子の動きを観察する (Priority: P1)

訪問者がカタログから二重振り子のデモページを開くと、二重振り子が物理法則に従って
動き始め、カオス的で予測不能な動きをリアルタイムに観察できる。開始・一時停止・
リセットの操作で観察を自由にコントロールできる。

**Why this priority**: 振り子が物理的に正しく動くことがこのデモの存在意義そのもの。
これ単体で「眺めて楽しめるデモ」として成立する MVP となる。

**Independent Test**: デモページを開き、振り子が自動または開始操作で動き出し、
二重振り子特有の不規則な軌道を描くこと、一時停止・リセットが機能することを
確認できれば、このストーリー単体で価値が検証できる。

**Acceptance Scenarios**:

1. **Given** デモページを開いた状態、**When** シミュレーションを開始する、
   **Then** 二重振り子が重力に従って揺れ始め、滑らかなアニメーションで描画される
2. **Given** シミュレーションが動作中、**When** 一時停止操作を行う、
   **Then** 振り子がその場で静止し、再開操作で続きから動き出す
3. **Given** シミュレーションが動作中、**When** リセット操作を行う、
   **Then** 振り子が初期位置に戻り、軌跡もクリアされる
4. **Given** カタログ一覧ページ、**When** シミュレーションカテゴリの当該エントリを選ぶ、
   **Then** 二重振り子デモページに遷移できる

---

### User Story 2 - パラメータを調整して挙動の変化を楽しむ (Priority: P2)

訪問者が 2 本の節（上腕・下腕）の長さと、それぞれの開始角度（初期位置）を
コントロールで調整し、条件の違いによって振る舞いが大きく変わる様子
（カオス性・初期値鋭敏性）を体験できる。

**Why this priority**: ユーザーが明示的に要望している操作機能であり、
「見るだけ」から「実験して遊べる」体験へ価値を引き上げる。ただし P1 の
シミュレーションが成立していることが前提のため P2 とする。

**Independent Test**: 各パラメータのコントロールを操作して値を変え、
リセット後の振り子の形状・動き出しが調整値どおりに変化することを確認できる。

**Acceptance Scenarios**:

1. **Given** デモページの設定パネル、**When** 上の節の長さを変更してリセットする、
   **Then** 振り子の上腕の見た目の長さが変わり、動きも変化する
2. **Given** デモページの設定パネル、**When** 各腕の開始角度を変更してリセットする、
   **Then** 振り子が指定した角度の位置から動き始める
3. **Given** いずれかのパラメータを範囲の上限・下限まで操作した状態、
   **When** シミュレーションを開始する、**Then** 破綻（描画乱れ・停止）なく動作する
4. **Given** シミュレーション動作中、**When** パラメータを変更する、
   **Then** 変更がいつ反映されるか（即時またはリセット時）が UI 上で明確にわかる

---

### User Story 3 - 美しい光跡の演出を楽しむ (Priority: P3)

振り子の先端（下の節の末端）が通った軌跡が光の尾のように画面へしばらく残り、
時間とともに滑らかに消えていく。カオス的な軌道が幾何学模様のような美しい
ビジュアルとして現れ、思わず見続けたくなる体験を提供する。

**Why this priority**: ユーザーが要望する「見た目の美しさ」の中核。ただし
シミュレーション（P1）と独立に追加できる視覚演出レイヤーのため P3 とする。

**Independent Test**: シミュレーションを一定時間動かし、先端の軌跡が残像として
表示されること、古い軌跡ほど薄くなり一定時間で消えることを確認できる。

**Acceptance Scenarios**:

1. **Given** シミュレーションが動作中、**When** 振り子の先端が移動する、
   **Then** 通過した経路に光跡が描画され、時間経過とともに徐々に薄れて消える
2. **Given** 光跡が表示された状態、**When** リセット操作を行う、
   **Then** 光跡もすべてクリアされる
3. **Given** 長時間（数分以上）動作させた状態、**When** 光跡が蓄積し続ける、
   **Then** 表示の滑らかさが維持される（古い光跡は破棄され、無限に蓄積しない）

---

### Edge Cases

- 節の長さや角度を範囲外の値にしようとした場合、入力は許容範囲内に制限される
- ブラウザのタブが非アクティブになり戻ってきた場合、シミュレーションが時間を
  一気にスキップして破綻することなく、自然に再開される
- 画面サイズが小さい端末では、描画領域とコントロールが重ならずに利用できる
- 長時間の連続動作でも数値計算が発散して振り子が画面外へ飛ぶようなことがない
- 「視差効果を減らす」（reduced motion）を OS で設定しているユーザーには、
  自動再生を控える等の配慮を行う

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはカタログ内に「シミュレーション」系のデモとして二重振り子
  ページを掲載し、カタログ一覧から到達できるようにしなければならない
- **FR-002**: システムは二重振り子（2 本の節が連結された振り子）の運動を物理法則に
  基づいてリアルタイムに計算し、アニメーション表示しなければならない
- **FR-003**: ユーザーは 2 本の節それぞれの長さを、定義された範囲内で調整できなければ
  ならない
- **FR-004**: ユーザーは 2 本の腕それぞれの開始角度（初期位置）を調整できなければ
  ならない
- **FR-005**: ユーザーはシミュレーションを開始・一時停止・リセットできなければ
  ならない。リセット時は現在のパラメータ設定で初期状態に戻る
- **FR-006**: システムは振り子の先端の軌跡を光跡として表示し、古い軌跡ほど薄く
  なって一定時間で消える表現にしなければならない
- **FR-007**: システムは初回表示時に、視覚的に興味深い動きが得られる既定の
  パラメータ値を設定していなければならない
- **FR-008**: ページはダークモード・ライトモードの双方で配色が破綻せず、
  モバイルからデスクトップまでの画面幅で利用できなければならない
- **FR-009**: 同一のパラメータ設定でリセットした場合、振り子は常に同じ初期状態から
  動き始めなければならない（再現性）

### Key Entities

- **シミュレーションパラメータ**: ユーザーが調整できる設定値の集合。節 1 の長さ、
  節 2 の長さ、腕 1 の開始角度、腕 2 の開始角度。それぞれに既定値と許容範囲を持つ
- **振り子の状態**: ある瞬間の振り子を表す値の集合。各腕の角度と角速度。
  パラメータと物理法則から時々刻々と更新される
- **光跡**: 振り子の先端が通過した位置の履歴。各点は経過時間に応じた濃度を持ち、
  一定時間経過後に破棄される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: シミュレーションのアニメーションが一般的な PC・スマートフォンで
  毎秒 60 コマ相当の滑らかさを維持し、目視でカクつきを感じない
- **SC-002**: 初めての訪問者が説明なしで、30 秒以内にパラメータを変更して
  振る舞いの変化を確認できる
- **SC-003**: デモページは表示開始から 3 秒以内に操作可能になる
- **SC-004**: 5 分間の連続動作後も表示の滑らかさと描画の正しさが維持される
- **SC-005**: 同一条件でのリセットを 10 回行っても、毎回同じ初期配置・同じ動き出しに
  なる（再現性 100%）
- **SC-006**: 光跡は最新の軌道がはっきり見え、古い軌道は滑らかに減衰して消える
  （残存時間はおおむね数秒のオーダー）

## Assumptions

- 調整できるパラメータは「2 本の節の長さ」と「2 本の腕の開始角度」に限定する。
  質量・重力加速度・摩擦などの調整は本バージョンの対象外とする（既定値で固定）
- 摩擦・空気抵抗は考慮しない理想系としてシミュレーションする（減衰しない）
- カタログ上は新しい「シミュレーション」カテゴリ（または同等の分類）に掲載する。
  カテゴリの新設が必要な場合はその追加も本機能の範囲に含む
- パラメータ変更の反映タイミングは「リセット（再スタート）時に適用」を基本とし、
  UI 上でそのことが分かるようにする
- シミュレーション結果の保存・共有（URL での状態共有、スクリーンショット等）は
  本バージョンの対象外とする
- 表示言語は既存カタログに合わせて日本語とする
- 効果音・BGM は対象外とする

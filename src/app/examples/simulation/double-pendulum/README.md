# 二重振り子シミュレーション

振り子の先にもう 1 つ振り子をつないだ「二重振り子」の物理シミュレーションです。
単純な構造なのに動きは予測不能なカオスになる、力学のもっとも有名な例のひとつを
ブラウザ上で再現しています。

## 学べること

### 1. 物理シミュレーションの基本構造（`_lib/double-pendulum.ts`）

二重振り子の運動は、4 つの数値（2 つの角度 θ1, θ2 と 2 つの角速度 ω1, ω2）で
完全に決まります。この「状態」を、ラグランジュ力学から導かれる運動方程式に
従って少しずつ時間発展させるのがシミュレーションの正体です。

物理エンジンは **React に依存しない純関数** として実装しています。
「同じ入力 → 必ず同じ出力」なので、ユニットテストが簡単に書け、
リセットの再現性も自然に成立します。

### 2. RK4 数値積分（なぜオイラー法ではダメか）

もっとも素朴なオイラー法（現在の傾きで一直線に進む）では、カオス系は誤差が
急速に蓄積し、エネルギーが増え続けて振り子が暴走します。このデモでは
**4 次ルンゲ＝クッタ法（RK4）** を使い、区間内の 4 か所で傾きを評価した
重み付き平均で進むことで、長時間でも安定した軌道を得ています。

`double-pendulum.test.ts` には「5 分相当（72,000 ステップ）積分しても
全力学的エネルギーの誤差が 1% 未満」というテストがあり、数値的な安定性を
機械的に保証しています。

### 3. 固定タイムステップ + アキュムレータ（`_components/double-pendulum-demo.tsx`）

`requestAnimationFrame` のフレーム間隔は毎回バラつきます。経過時間をそのまま
物理計算に使うと結果がフレームレート依存になり、カオス系では再現性が失われます。

そこで「経過時間を貯金（accumulator）し、固定の刻み幅（1/240 秒）で貯金が
なくなるまで積分する」という定番パターンを使っています。ゲーム開発でも
広く使われる手法です。タブ復帰時の時間スキップ対策（delta のクランプ）も
セットで実装しています。

### 4. React と 60fps アニメーションの共存

毎フレーム `setState` すると再レンダリングのコストで 60fps 維持が難しくなります。
このデモでは:

- **UI に反映される値**（パラメータ、再生状態）→ React の `useState`
- **毎フレーム変わる値**（力学状態、光跡）→ `useRef` + Canvas 直接描画

と分離することで、React のレンダリングサイクルの外で描画ループを回しています。

### 5. 光跡（トレイル）の表現（`_lib/trail.ts` + `_components/pendulum-canvas.tsx`)

先端の位置を上限付きのバッファに記録し、「太い半透明の線」と「細い明るい線」の
二重描きで発光感を出しています。各点は **シミュレーション時刻** を持つため、
一時停止中はフェードも止まります（実時間基準だと一時停止中に光跡が消えて
しまいます）。

```
    ├── double-pendulum.test.ts
    ├── trail.ts                    # 光跡バッファ（純関数）
    └── trail.test.ts
```

## 遊び方のヒント

- 開始角度をほんの 1° だけ変えてリセットしてみてください。最初の数秒は
  同じように見えて、すぐにまったく違う軌道になります（初期値鋭敏性）
- 両方の腕を 180° 近くまで上げると、エネルギーが大きく激しい回転運動になります
- 節 1 を短く・節 2 を長くすると、先端の光跡が大きく繊細な模様を描きます

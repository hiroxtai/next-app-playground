# ADR (Architecture Decision Record)

このディレクトリには、プロジェクトの重要な設計判断を記録する ADR を置く。

## 運用ルール

- ファイル名: `NNNN-<英語ケバブケース>.md`（NNNN は 4 桁の連番）
- テンプレート: [0000-template.md](./0000-template.md)
- Status の遷移: `Proposed` → `Accepted` →（置き換えられた場合）`Superseded by ADR-NNNN`
- 一度 Accepted になった ADR は編集しない。決定を変える場合は新しい ADR を作成し、旧 ADR の Status を更新する

## ADR を書く基準

以下のいずれかに該当する決定は ADR に記録する:

- アーキテクチャ・構成の変更（レイヤ構造、状態管理方式、ディレクトリ設計など）
- 技術選定(ライブラリ・フレームワーク・ツールの採用/置換/廃止)
- 横断的な規約の導入・変更（テスト方針、エラーハンドリング方針など）
- セキュリティ・パフォーマンスに関わるトレードオフの選択

バグ修正・軽微な機能追加・依存バージョン更新は記録しない。

## 自動化

PR がマージされると `claude-adr.yml` ワークフローが発火し、ADR に値する設計判断が
含まれる場合は `write-adr` スキルが ADR の PR を自動作成する。
手動で作成する場合もテンプレートと上記ルールに従うこと。
